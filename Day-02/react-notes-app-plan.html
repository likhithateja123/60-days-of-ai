Section 01

Stack choices, confirmed & justified

Layer Choice Justification
Build tool Vite Sub-second HMR, native ESM, PWA plugin support — far faster DX than CRA.
UI framework React 18 Concurrent features (useTransition) improve search UX; massive ecosystem.
State management Zustand Minimal boilerplate, no context hell, built-in persist middleware — ideal for notes state. Redux is overkill here.
Persistence (local) IndexedDB via Dexie.js Handles large documents better than localStorage; Dexie adds a clean Promise API. Zustand persist syncs in parallel.
Rich text Tiptap v2 ProseMirror-based, headless, markdown shortcuts built-in, fully accessible. Lighter than Quill; more extensible than react-markdown alone.
Routing React Router v6 Nested routes map naturally to notebooks → notes hierarchy. Data loaders simplify note fetching.
Styling Tailwind CSS v3 Utility-first = fast iteration; purge keeps bundle tiny; dark mode via class strategy is trivial.
Search Fuse.js Client-side fuzzy search; zero server cost; instantaneous results. No Algolia needed for MVP.
Offline / PWA vite-plugin-pwa Generates service worker + manifest automatically; Workbox handles caching strategies.
Auth (optional) Supabase Auth Free tier, Row-Level Security, pairs with Supabase Postgres for cloud sync in Phase 3.
Testing Vitest + Testing Library Vitest shares Vite config; Testing Library enforces a11y-first queries.
Deployment Vercel Zero-config deploy, automatic preview URLs per PR, free tier sufficient.

Zustand store shape

typescriptinterface Note {
  id:         string;
  title:      string;
  content:    string;   // HTML from Tiptap
  tags:       string[];
  notebookId: string | null;
  createdAt:  number;
  updatedAt:  number;
  deletedAt:  number | null;  // soft delete
}

interface NotesStore {
  notes:       Note[];
  addNote:     (partial: Partial<Note>) => void;
  updateNote:  (id: string, patch: Partial<Note>) => void;
  deleteNote:  (id: string) => void;
  restoreNote: (id: string) => void;
}
Section 05

Code snippets for core features

CRUD — Zustand store with Dexie persistence

useNotesStore.tsimport { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'
import { db } from './db'  // Dexie instance

export const useNotesStore = create<NotesStore>()(persist(
  (set, get) => ({
    notes: [],

    addNote: (partial) => {
      const note: Note = {
        id: crypto.randomUUID(),
        title: '', content: '', tags: [],
        notebookId: null, deletedAt: null,
        createdAt: Date.now(), updatedAt: Date.now(),
        ...partial
      }
      set((s) => ({ notes: [note, ...s.notes] }))
      db.notes.put(note)  // write-through to IndexedDB
    },

    updateNote: (id, patch) => {
      set((s) => ({
        notes: s.notes.map(n =>
          n.id === id ? { ...n, ...patch, updatedAt: Date.now() } : n
        )
      }))
      db.notes.update(id, { ...patch, updatedAt: Date.now() })
    },

    deleteNote: (id) => get().updateNote(id, { deletedAt: Date.now() }),
  }),
  { name: 'notes-store' }
))

Autosave — debounced update hook

useAutosave.tsimport { useEffect, useRef } from 'react'

export function useAutosave(
  value: string,
  onSave: (v: string) => void,
  delay = 500
) {
  const timer = useRef<ReturnType<typeof setTimeout>>()

  useEffect(() => {
    timer.current && clearTimeout(timer.current)
    timer.current = setTimeout(() => onSave(value), delay)
    return () => clearTimeout(timer.current)
  }, [value, onSave, delay])
}

Search — Fuse.js with useTransition

useSearch.tsimport Fuse from 'fuse.js'
import { useMemo, useState, useTransition } from 'react'
import { useNotesStore } from './useNotesStore'

export function useSearch() {
  const notes   = useNotesStore(s => s.notes.filter(n => !n.deletedAt))
  const [query, setQuery] = useState('')
  const [isPending, startTransition] = useTransition()

  const fuse = useMemo(() => new Fuse(notes, {
    keys: ['title', 'content', 'tags'],
    threshold: 0.35, includeScore: true
  }), [notes])

  const results = useMemo(() =>
    query ? fuse.search(query).map(r => r.item) : notes
  , [query, fuse, notes])

  const search = (q: string) =>
    startTransition(() => setQuery(q))

  return { results, query, search, isPending }
}

Tiptap rich text editor component

TiptapEditor.tsximport { useEditor, EditorContent } from '@tiptap/react'
import StarterKit from '@tiptap/starter-kit'
import { useAutosave } from './useAutosave'

export function TiptapEditor({
  noteId, initialContent, onUpdate
}: {
  noteId: string
  initialContent: string
  onUpdate: (html: string) => void
}) {
  const editor = useEditor({
    extensions: [StarterKit],
    content: initialContent,
    onUpdate: ({ editor }) => onUpdate(editor.getHTML()),
  })

  return (
    <EditorContent
      editor={editor}
      className="prose prose-sm max-w-none focus:outline-none"
      aria-label="Note body"
      aria-multiline="true"
    />
  )
}

Section 07

Accessibility, responsiveness & testing

Accessibility

  • All interactive elements reachable by keyboard; visible focus ring via Tailwind focus-visible:ring
  • ARIA roles on sidebar nav, note list (role="list"), editor (role="textbox" aria-multiline), search (role="combobox")
  • Tiptap exposes aria-label on the content-editable region
  • Color contrast ≥4.5:1 for all text; tested with axe DevTools
  • Reduced motion: wrap animations in @media (prefers-reduced-motion: reduce)
  • Skip-to-content link at page top

Responsiveness

  • Mobile (<640px): single-pane, sidebar slides in as drawer
  • Tablet (640–1024px): collapsible sidebar (icon-only or expanded)
  • Desktop (>1024px): persistent 3-pane (notebooks + list + editor)
  • Fluid typography using Tailwind's text-sm/base/lg breakpoints
